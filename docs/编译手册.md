## 1. 概述

Cocoa 是一个基于 C# 13 (.NET 9.0 / CLR) 的方言编译器，支持：

- 直接编译 `.co` 源文件为 .NET 可执行文件或库
- 通过 `.coproj` 项目文件组织多源文件项目
- 通过 `.cosln` 解决方案文件组织多项目解决方案
- 交互式 REPL 环境
- 调用任意 .NET API（通过 `-r` 引用程序集）


### 工具链组成

| 工具 | 文件 | 用途 |
|------|------|------|
| `coc` | `src/Cocoa.Compiler/` | 命令行编译器 |
| `coi` | `src/Cocoa.Interactive/` | 交互式 REPL |
| `coc.exe` | `src/coc.cmd` | 便捷调用脚本 |
| `coi.exe` | `src/coi.cmd` | 便捷调用脚本 |

## 2. 安装与构建

### 2.1 前置要求

- .NET 9.0 SDK
- 操作系统：Windows / Linux / macOS

### 2.2 构建编译器

```bash
# Windows
src\build.cmd

# Linux/macOS
src/build.sh
```

构建完成后，编译器可执行文件位于 `src/Cocoa.Compiler/bin/Debug/net9.0/coc.exe`。

### 2.3 验证安装

```bash
coc --help
```

输出应显示编译器选项说明。

## 3. 编译器命令行 (`coc`)

### 3.1 基本用法

```bash
coc <source-files> [options]
```

### 3.2 选项说明

| 选项 | 说明 | 示例 |
|------|------|------|
| `<source-files>` | Cocoa 源文件路径（支持 `.co`、`.coproj`、`.cosln`） | `hello.co` |
| `-o <path>` | 输出程序集路径（默认：第一个源文件名改后缀 `.exe`） | `-o output.exe` |
| `-m <name>` | 模块名称（默认：输出文件名去后缀） | `-m MyApp` |
| `-r <path>` | 引用程序集路径（可多次使用） | `-r System.dll` |
| `--debug` | 调试模式（默认）：生成 PDB 符号文件，禁用 JIT 优化 | `coc hello.co --debug` |
| `--release` | 发布模式：不生成 PDB，启用 JIT 优化 | `coc hello.co --release` |
| `-?` / `-h` / `--help` | 显示帮助信息 | `coc -h` |

### 3.3 编译单文件

```bash
# 编译 hello.co → hello.exe
coc hello.co

# 编译 hello.co → output.exe
coc hello.co -o output.exe

# 编译 hello.co 并引用额外的程序集
coc hello.co -r /path/to/MyLib.dll
```

### 3.4 编译多源文件

```bash
# 同时编译多个 .co 文件
coc main.co utils.co math.co

# 指定输出
coc main.co utils.co -o myapp.exe -m MyApp
```

### 3.5 使用项目文件编译

```bash
# 加载 .coproj 项目文件（自动读取 sources/references 等配置）
coc MyApp.coproj

# 覆盖项目文件中的设置
coc MyApp.coproj -o custom.exe -r extra.dll
```

### 3.6 使用解决方案文件编译

```bash
# 加载 .cosln 解决方案文件（依次编译所有项目）
coc MyApp.cosln

# 解决方案中的项目依赖关系自动处理
```

### 3.7 完整示例

```bash
# 单文件 + 引用 .NET 程序集
coc hello.co -r System.Runtime.dll -o hello.exe

# 项目文件
coc src/MyApp/MyApp.coproj -o bin/Release/MyApp.exe

# 解决方案（批量编译）
coc src/MyApp.cosln
```

## 4. 项目文件 (`.coproj`)

### 4.1 格式说明

`.coproj` 是 Cocoa 项目的配置文件，采用 **JSON** 格式（与 MSBuild `.csproj` 无关，是 Cocoa 原生格式）。

### 4.2 文件结构

```json
{
    "name": "MyProject",
    "outputType": "Exe",
    "targetFramework": "net9.0",
    "sources": [
        "*.co"
    ],
    "references": [
        "System.Runtime.dll",
        "System.Console.dll"
    ],
    "outputPath": "bin/Debug/net9.0/",
    "moduleName": "MyProject",
    "entryPoint": "Main"
}
```

### 4.3 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `name` | `string` | 否 | 输出文件名 | 项目名称 |
| `outputType` | `string` | 否 | `"Exe"` | `"Exe"` 可执行文件、`"Library"` 库 |
| `sources` | `string[]` | 否 | `["*.co"]` | 源文件 glob 模式列表 |
| `references` | `string[]` | 否 | `[]` | 直接程序集引用路径 |
| `projectReferences` | `string[]` | 否 | `[]` | 项目引用路径（.coproj），自动解析输出程序集 |
| `packageReferences` | `object` | 否 | `{}` | NuGet 包引用，key=包名，value=版本 |
| `outputPath` | `string` | 否 | `"."` | 输出目录 |
| `moduleName` | `string` | 否 | `name` 字段 | 模块名称 |
| `entryPoint` | `string` | 否 | `"Main"` | 入口函数名（仅 `Exe`） |
| `debug` | `bool` | 否 | `true` | 调试模式（PDB+非优化） |
| `define` | `string[]` | 否 | `[]` | 条件编译符号列表 |
| `deterministic` | `bool` | 否 | `true` | 确定性构建（相同输入→相同输出） |
| `treatWarningsAsErrors` | `bool` | 否 | `false` | 警告视为错误 |
| `version` | `string` | 否 | `"1.0.0.0"` | 程序集版本号 |
| `fileVersion` | `string` | 否 | `version` | 文件版本号 |
| `company` | `string` | 否 | `""` | 公司名称 |
| `copyright` | `string` | 否 | `""` | 版权声明 |
| `description` | `string` | 否 | `""` | 程序集描述 |

### 4.4 项目文件示例

**控制台应用** `Hello.coproj`：

```json
{
    "name": "Hello",
    "outputType": "Exe",
    "debug": true,
    "sources": [
        "*.co"
    ],
    "references": [
        "System.Runtime.dll",
        "System.Console.dll"
    ]
}
```

**类库** `MyLib.coproj`：

```json
{
    "name": "MyLib",
    "outputType": "Library",
    "sources": [
        "src/**/*.co"
    ],
    "references": [
        "System.Runtime.dll"
    ]
}
```

### 4.5 文件 glob 模式

| 模式 | 匹配 |
|------|------|
| `*.co` | 当前目录所有 `.co` 文件 |
| `src/**/*.co` | `src/` 及其子目录下所有 `.co` 文件 |
| `main.co` | 单文件 |
| `["a.co", "b.co"]` | 多个指定文件 |

## 5. 解决方案文件 (`.cosln`)

### 5.1 格式说明

`.cosln` 是 Cocoa 解决方案的配置文件，同样采用 JSON 格式。

### 5.2 文件结构

```json
{
    "name": "MyApp",
    "projects": [
        "src/MyLib/MyLib.coproj",
        "src/MyApp/MyApp.coproj"
    ]
}
```

### 5.3 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `name` | `string` | 否 | 解决方案文件名 | 解决方案名称 |
| `projects` | `string[]` | 是 | — | 项目文件路径列表 |
| `folders` | `object[]` | 否 | `[]` | 解决方案文件夹，每个含 `name` + `projects` |
| `configurations` | `object` | 否 | `{"Debug":...,"Release":...}` | 自定义解决方案配置 |
| `projectConfig` | `object` | 否 | `{}` | 每个项目在特定配置下的行为覆盖 |

### 5.4 文件夹结构

```json
{
    "name": "src",
    "projects": [ "src/Core/Core.coproj", "src/WebApi/WebApi.coproj" ]
}
```

文件夹仅用于 IDE 逻辑组织，不影响编译行为。项目可出现在多个文件夹中。

### 5.5 项目配置覆盖

```json
{
    "projectConfig": {
        "tests/MyApp.Tests.coproj": {
            "Release": { "build": false }
        }
    }
}
```

`build: false` 表示该配置下跳过编译（如测试项目在 Release 下不编译）。

### 5.6 编译顺序

编译顺序由 `projectReferences` 依赖图**拓扑排序**决定，不再依赖数组顺序。无依赖时维持数组顺序。

### 5.7 解决方案文件示例

```json
{
    "name": "EnterpriseApp",
    "projects": [
        "src/Core/Core.coproj",
        "src/DataAccess/DataAccess.coproj",
        "src/WebApi/WebApi.coproj",
        "tests/UnitTests.coproj"
    ],
    "folders": [
        { "name": "src", "projects": ["src/Core/Core.coproj", "src/DataAccess/DataAccess.coproj", "src/WebApi/WebApi.coproj"] },
        { "name": "tests", "projects": ["tests/UnitTests.coproj"] }
    ],
    "projectConfig": {
        "tests/UnitTests.coproj": {
            "Release": { "build": false }
        }
    }
}
```

编译顺序：Core → DataAccess → WebApi（根据 projectReferences 推断），UnitTests（Release 下跳过）。

## 6. 交互式环境 (`coi`)

### 6.1 启动 REPL

```bash
coi
```

### 6.2 REPL 命令

| 命令 | 说明 |
|------|------|
| `#help` | 显示帮助 |
| `#exit` | 退出 REPL |
| `#r "path"` | 引用程序集 |
| `#load "file.co"` | 加载源文件 |
| `#submit` | 提交并执行当前输入 |

### 6.3 REPL 示例

```
> let x = 42
> print(x)
42
> #r "System.Runtime.dll"
> System.Math.Max(10, 20)
20
```

### 6.4 从 stdin 执行脚本

```bash
echo "print('hello')" | coi
```

## 7. Cocoa 语言快速入门

### 7.1 Hello World

```cocoa
// hello.co
function main()
{
    print("What's your name?")
    let name = input()
    print("Hello " + name + "!")
}
```

编译运行：

```bash
coc hello.co
./hello.exe
```

### 7.2 变量与类型

```cocoa
// 变量声明
let x = 42              // 不可变
var y = 10              // 可变
let name: string = "Alice"
var count: int = 0

// 类型
let a: int = 42
let b: double = 3.14
let c: bool = true
let d: string = "hello"
```

### 7.3 控制流

```cocoa
// if/else
if (x > 0)
{
    print("positive")
}
else if (x == 0)
{
    print("zero")
}
else
{
    print("negative")
}

// while
var i = 0
while (i < 10)
{
    print(i)
    i = i + 1
}

// for-range
for (var i = 0 to 5)
{
    print(i)
}

// for C 风格
for (var i = 0; i < 10; i++)
{
    print(i)
}

// do-while
do
{
    print("once")
} while (false)
```

### 7.4 函数

```cocoa
function add(a: int, b: int): int
{
    return a + a + b
}

function greet(name: string)
{
    print("Hello, " + name)
}

// 递归
function factorial(n: int): int
{
    if (n <= 1)
    {
        return 1
    }
    return n * factorial(n - 1)
}
```

### 7.5 调用 .NET API（通过 `-r` 引用）

```cocoa
// 编译时需要 -r System.Runtime.dll
System.Console.WriteLine("hello from Cocoa")
let max = System.Math.Max(10, 20)
```

## 8. 常见问题

### 8.1 编译器报 "file doesn't exist"

检查源文件路径是否正确。支持相对路径和绝对路径。

### 8.2 引用程序集路径

- 引用的 DLL 必须是有效的 .NET 程序集
- 使用绝对路径或相对于工作目录的路径
- 常用运行时程序集位于 .NET SDK 目录下

### 8.3 输出文件

默认情况下，编译器生成：
- `.exe` 或 `.dll` — 编译后的程序集
- `.pdb` — 调试符号文件（含源代码映射）
