## 1. 项目概览

### 1.1 解决方案结构

| 项目 | 输出 | 用途 |
|------|------|------|
| `Cocoa` | 类库 | 核心编译器库（词法 → 语法 → 语义 → 降低 → 解释/发射） |
| `Cocoa.Compiler` | `coc.exe` | 命令行编译器 |
| `Cocoa.Interactive` | `coi.exe` | 交互式 REPL |
| `Cocoa.Tests` | 测试库 | xUnit 单元测试 |
| `Cocoa.Generators` | 源生成器 | Roslyn 源生成器，自动生成 `GetChildren()` |

### 1.2 编译管线

```
SourceText (字符序列 + 行列映射)
  → Lexer (字符 → Token)
  → Parser (Token → 语法树 AST)
  → Binder (AST → Bound Tree + 符号解析 + 类型检查)
  → Lowerer (高层 Bound Tree → goto-based IR)
  → Evaluator (树遍历解释执行)  或  Emitter (Mono.Cecil → .exe + .pdb)
```

### 1.3 NuGet 依赖

| 包名 | 版本 | 用途 |
|------|------|------|
| `Mono.Cecil` | 0.11.6 | IL 生成与程序集写入 |
| `Mono.Options` | 6.12.0 | CLI 参数解析 |
| `System.Collections.Immutable` | 9.0.9 | 不可变集合 |

## 4. 运算符

### 4.1 一元运算符（优先级 6）

| 语法 | Token | Bound 类型 | 操作数 | 结果 | 运算 |
|------|-------|-----------|--------|------|------|
| `+x` | `+` | `Identity` | `int` | `int` | 恒等 |
| `-x` | `-` | `Negation` | `int` | `int` | 算术取反 |
| `!x` | `!` | `LogicalNegation` | `bool` | `bool` | 逻辑非 |
| `~x` | `~` | `OnesComplement` | `int` | `int` | 按位取反 |

### 4.2 二元运算符

**优先级 5（乘除）：**

| 语法 | Token | Bound 类型 | 左类型 | 右类型 | 结果 |
|------|-------|-----------|--------|--------|------|
| `x * y` | `*` | `Multiplication` | `int` | `int` | `int` |
| `x / y` | `/` | `Division` | `int` | `int` | `int` |

**优先级 4（加减）：**

| 语法 | Token | Bound 类型 | 左类型 | 右类型 | 结果 |
|------|-------|-----------|--------|--------|------|
| `x + y` | `+` | `Addition` | `int` | `int` | `int` |
| `x + y` | `+` | `Addition` | `string` | `string` | `string` |
| `x - y` | `-` | `Subtraction` | `int` | `int` | `int` |

**优先级 3（关系）：**

| 语法 | 左类型 | 结果 | 说明 |
|------|--------|------|------|
| `x == y` | `int`/`bool`/`string`/`any` | `bool` | 相等比较 |
| `x != y` | `int`/`bool`/`string`/`any` | `bool` | 不等比较 |
| `x < y` `x <= y` `x > y` `x >= y` | `int` | `bool` | 大小比较 |

**优先级 2（AND）：**

| 语法 | Token | 左类型 | 结果 |
|------|-------|--------|------|
| `x & y` | `&` | `int`/`bool` | `int`/`bool` |
| `x && y` | `&&` | `bool` | `bool`（逻辑与） |

**优先级 1（OR/XOR）：**

| 语法 | Token | 左类型 | 结果 |
|------|-------|--------|------|
| `x \| y` | `\|` | `int`/`bool` | `int`/`bool` |
| `x \|\| y` | `\|\|` | `bool` | `bool`（逻辑或） |
| `x ^ y` | `^` | `int`/`bool` | `int`/`bool` |

> 注意：`&&` 和 `||` **未使用短路求值**（Emitting 时使用 `and`/`or` IL 指令，而非常用的短路模式），Evaluator 中正确实现短路。

### 4.3 赋值运算符

| 语法 | Token | 类型 |
|------|-------|------|
| `x = y` | `=` | 简单赋值 |
| `x += y` | `+=` | 复合赋值 |
| `x -= y` | `-=` | 复合赋值 |
| `x *= y` | `*=` | 复合赋值 |
| `x /= y` | `/=` | 复合赋值 |
| `x &= y` | `&=` | 复合赋值 |
| `x \|= y` | `\|=` | 复合赋值 |
| `x ^= y` | `^=` | 复合赋值 |

复合赋值在 Lowerer 中降低为 `x = (x op y)`。

### 4.4 类型转换

| 从 | 到 | 方式 | 说明 |
|----|----|------|------|
| `T` | `T` | 恒等转换 | 同一类型 |
| `T`（非 void） | `any` | 隐式转换 | 拓宽到 object |
| `any` | `T`（非 void） | 显式转换 | 向下转换，需 `type(expr)` 语法 |
| `bool` | `string` | 显式转换 | `string(x)` |
| `int` | `string` | 显式转换 | `string(x)` |
| `string` | `bool` | 显式转换 | `bool(x)` |
| `string` | `int` | 显式转换 | `int(x)` |
| 其他 | 其他 | 无（错误） | 编译报错 |

## 8. 词法分析 (Lexer)

### 8.1 识别的 Token（52 种）

**标点符号：** `+` `-` `*` `/` `(` `)` `{` `}` `:` `,` `~` `^` `&` `|` `=` `!` `<` `>`

**复合标点：** `+=` `-=` `*=` `/=` `^=` `&=` `|=` `&&` `||` `==` `!=` `<=` `>=`

**字面量：** `NumberToken`（整数）、`StringToken`（双引号字符串）、`IdentifierToken`（标识符）

**关键字：** 14 个（见 §3）

**特殊：** `BadToken`（非法字符）、`EndOfFileToken`（文件结束）

### 8.2 字符串字面量

- 双引号包裹 `"..."` 
- 未终止字符串报错
- 转义方式：双引号 `""` 表示一个 `"` 字符

### 8.3 数字字面量

- 仅整数（十进制数字 0-9）
- `int.TryParse` 解析，溢出/格式非法报错

### 8.4 标识符

- 以字母或 `_` 开头，后续可包含字母、数字、`_`
- 关键字匹配通过 `SyntaxFacts.GetKeywordKind` 完成

### 8.5 次要文本（Trivia）

| 类型 | 说明 |
|------|------|
| `WhitespaceTrivia` | 空格、制表符等 |
| `LineBreakTrivia` | `\r` `\n` `\r\n` |
| `SingleLineCommentTrivia` | `//` 到行尾 |
| `MultiLineCommentTrivia` | `/* ... */`（不支持嵌套） |

- 前置次要文本附加到后续 Token
- 后置次要文本（仅换行）附加到前一个 Token
- 注释在 `ReadTrivia` 阶段解析

## 9. 语法分析 (Parser)

### 9.1 解析方法（21 个）

**顶级：**
- `ParseCompilationUnit` → `CompilationUnitSyntax`
- `ParseMembers` / `ParseMember` → 成员列表

**函数：**
- `ParseFunctionDeclaration` → `FunctionDeclarationSyntax`
- `ParseParameterList` / `ParseParameter` → 参数列表

**语句（ParseStatement 派发）：**
```
{           → BlockStatement
let / var   → VariableDeclaration
if          → IfStatement
while       → WhileStatement
do          → DoWhileStatement
for         → ForStatement
break       → BreakStatement
continue    → ContinueStatement
return      → ReturnStatement
default     → ExpressionStatement
```

**表达式解析链：**
```
ParseExpression
  → ParseAssignmentExpression
    → ParseBinaryExpression (Pratt 解析器，优先级爬升)
      → ParsePrimaryExpression
        → (    → ParenthesizedExpression
        → bool → BooleanLiteral
        → num → NumberLiteral
        → str → StringLiteral
        → id  → NameOrCallExpression
          → id( → CallExpression
          → id  → NameExpression
```

### 9.2 运算符优先级（高到低）

| 优先级 | 运算符 |
|--------|--------|
| 6 | 一元: `+` `-` `!` `~` |
| 5 | 乘除: `*` `/` |
| 4 | 加减: `+` `-` |
| 3 | 关系: `==` `!=` `<` `<=` `>` `>=` |
| 2 | AND: `&` `&&` |
| 1 | OR/XOR: `\|` `\|\|` `^` |

### 9.3 错误处理

- `MatchToken`：期望 Token 不存在时报错并产生"缺失 Token"
- 无限循环保护：若当前 parse 未消耗 Token，则强制前进
- 词法坏 Token 吸收为 `SkippedTextTrivia`

## 10. 语义分析 (Binder)

### 10.1 绑定方法（22+ 个）

| 方法 | 返回 | 说明 |
|------|------|------|
| `BindGlobalScope` | `BoundGlobalScope` | 全局作用域入口（第一遍） |
| `BindProgram` | `BoundProgram` | 完整程序绑定（第二遍） |
| `BindStatementInternal` | `BoundStatement` | 语句派发 |
| `BindExpressionInternal` | `BoundExpression` | 表达式派发 |
| `BindFunctionDeclaration` | (副作用) | 注册函数到作用域 |
| `BindVariableDeclaration` | `BoundVariableDeclaration` | 变量声明绑定 |
| `BindConversion` | `BoundExpression` | 类型转换插入 |

### 10.2 作用域规则

- `BoundScope` 带父链（词法作用域）
- 函数和变量在当前作用域声明，内层作用域可见外层
- 内置函数预声明在根作用域
- REPL 提交通过 `Previous` 编译链累积符号

### 10.3 验证规则

- 禁止重复参数名
- 禁止同一作用域重复声明
- `main` 函数必须为 `(): void`
- 不能混用 `main` 函数和全局语句
- `break`/`continue` 仅能在循环中使用
- `let` 变量不可赋值
- 返回值类型匹配（void vs 值）
- 非 void 函数所有代码路径必须返回
- 表达式语句必须为赋值、复合赋值或调用
- 只含 `let var = expr;` 的变量声明中，右侧表达式类型与变量类型匹配

## 11. 中间表示降低 (Lowerer)

### 11.1 降低变换

| 变换 | 输入 → 输出 |
|------|-------------|
| If 语句 | → `gotoFalse condition elseLabel;` then块 `goto endLabel;` elseLabel: else块 endLabel: |
| While 循环 | → `goto continue;` body: 循环体 continue: `gotoTrue condition body;` break: |
| Do-While | → body: 循环体 continue: `gotoTrue condition body;` break: |
| For 循环 | → `{ var i = lower; let ub = upper; while (i <= ub) { body; i = i + 1; } }` |
| 复合赋值 | → `a = (a op b)` |
| 变量声明 | → 包裹 `BoundSequencePointStatement` |
| 表达式语句 | → 包裹 `BoundSequencePointStatement` |
| 条件 goto | → 常量折叠（条件恒定则替换为 Goto 或 Nop） |

### 11.2 降低后 IR

- `BoundLabelStatement` — 标签定义
- `BoundGotoStatement` — 无条件跳转
- `BoundConditionalGotoStatement` — 条件跳转
- `BoundNopStatement` — 空操作
- `BoundSequencePointStatement` — 调试位置标记

### 11.3 控制流图 (ControlFlowGraph)

- `BasicBlock` — 单入单出的指令序列
- `BasicBlockBranch` — 块间带条件的有向边
- `GraphBuilder` — 基于标签/goto/return 切分块
- `AllPathsReturn` — Binder 用于验证所有路径返回
- 支持 DOT 图形输出（`WriteTo`）

## 12. 解释执行 (Evaluator)

### 12.1 语句求值

| 语句 | 行为 |
|------|------|
| `NopStatement` | 跳过 |
| `VariableDeclaration` | 求值初始值，赋值给本地/全局变量 |
| `ExpressionStatement` | 求值表达式，丢弃结果 |
| `GotoStatement` | 跳转到标签对应指令索引 |
| `ConditionalGotoStatement` | 条件满足时跳转 |
| `LabelStatement` | 跳过（跳转目标） |
| `ReturnStatement` | 求值表达式，返回值 |

### 12.2 表达式求值

| 表达式 | 行为 |
|--------|------|
| `LiteralExpression` | 返回常量值 |
| `VariableExpression` | 从本地栈或全局查找变量 |
| `AssignmentExpression` | 求值 RHS，赋值，返回值 |
| `UnaryExpression` | 求值操作数，应用运算符 |
| `BinaryExpression` | 求值左右操作数，应用运算符 |
| `CallExpression` | 内置函数调用或用户函数调用 |
| `ConversionExpression` | 求值表达式，转换类型 |

### 12.3 常量折叠

在 Evaluator 和 Emitter 中，如果 BoundExpression 的 `ConstantValue` 不为 null，则直接使用常量值，跳过运行时求值路径。

## 13. IL 生成 (Emitter)

### 13.1 语句 IL 模式

| 语句 | IL 指令 |
|------|---------|
| `NopStatement` | `nop` |
| `VariableDeclaration` | `[init]; stloc` |
| `LabelStatement` | 记录指令索引 |
| `GotoStatement` | `br`（前向修复） |
| `ConditionalGotoStatement` | `brtrue` / `brfalse` |
| `ReturnStatement` | `[expr]; ret` |
| `ExpressionStatement` | `expr; pop` |
| `SequencePointStatement` | 内部语句 + 调试序列点 |

### 13.2 表达式 IL 模式

| 表达式 | IL 指令 |
|--------|---------|
| 布尔常量 | `ldc.i4.0` / `ldc.i4.1` |
| 整数常量 | `ldc.i4 <value>` |
| 字符串常量 | `ldstr <value>` |
| 参数变量 | `ldarg <ordinal>` |
| 本地变量 | `ldloc <var>` |
| 赋值 | `expr; dup; stloc <var>` |
| 一元恒等 | (无操作) |
| 一元取反 | `neg` |
| 一元逻辑非 | `ldc.i4.0; ceq` |
| 一元补码 | `not` |
| 整数加减乘除 | `add`/`sub`/`mul`/`div` |
| 字符串拼接 | `call String.Concat` |
| 位与/或/异或 | `and`/`or`/`xor` |
| 整数相等/不等 | `ceq` / `ceq; ldc.i4.0; ceq` |
| 字符串相等/不等 | `call object.Equals` |
| 大小比较 | `clt` / `cgt` |
| `print` | `call Console.WriteLine` |
| `input` | `call Console.ReadLine` |
| `random` | `call Random.get_Shared; callvirt Random.Next` |
| 用户函数 | `call <method>` |
| 转换到 any | `box`（值类型时） |
| 转换到 bool/int/string | `box; call Convert.ToXxx` |

### 13.3 程序集输出

- 输出：.NET PE 程序集 (.exe) + 可移植 PDB
- 入口点设置为 `main` 函数（或脚本入口）
- `DebuggableAttribute` 根据调试/发布模式设置
- 字符串拼接优化：扁平化 `+` 树，折叠相邻常量，选择合适的 `String.Concat` 重载

## 14. 编译管线 (Compilation)

### 14.1 编译流程

```
SourceText
  → SyntaxTree.Parse (Lexer + Parser) → CompilationUnitSyntax + diagnostics
  → Compilation.Create / CreateScript
    → Binder.BindGlobalScope (第一遍) → BoundGlobalScope + 函数/变量
    → Binder.BindProgram (第二遍) → BoundProgram + 降低后的函数体
  → Evaluation (Evaluator.Evaluate) — 直接解释
  → Emission (Emitter.Emit) — IL → .exe 程序集
```

### 14.2 两种编译模式

| 模式 | 入口 | 用途 |
|------|------|------|
| `Create` | `main()` 函数或全局语句 | 产生 .exe 程序集 |
| `CreateScript` | 累积提交 | REPL 增量编译，可返回值 |
