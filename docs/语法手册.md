## 1. 程序结构

一个 Cocoa 程序由一个或多个源文件组成。每个源文件可以包含：

- `using` 导入指令
- 命名空间声明（内含类型、函数声明等）
- 全局语句（脚本模式）
- 函数声明
- 类声明
- 接口声明
- 枚举声明

**全局语句模式**：
```
print("Hello World")
```

**函数入口模式**：
```
public static Main(): void
{
    print("Hello World")
}
```

也支持带命令行参数的变体：
```
public static Main(args: string[]): void
{
    print(args[0])
}
```

**两种模式互斥**：一个程序不能同时包含全局语句和 `Main` 函数。

**命名空间模式**：
```
using System

namespace MyApp
{
    public class Program
    {
        public static Main(): void
        {
            Console.WriteLine("hello")
        }
    }
}
```

### 10.6 For 语句

Cocoa 支持两种 for 循环语法。

#### 10.6.1 C# 风格 `for-range` (🔧)

```
// 带 var 声明（推荐）
for (var i = 0 to 10)
{
    print(i)
}

// 省略 var（变量需已在作用域内）
for (i = 0 to 10)
{
    print(i)
}
```

循环变量 `i` 在每次迭代开始时自动递增。括号内只能用 `var`（变量可变），不能用 `let`（不可变）。省略 `var` 时，`i` 必须在外部作用域已定义，否则编译器报错。

#### 10.6.2 C 风格 `for (init; cond; update)` (🔧)

```
for (var i = 0; i < 100; i++)
{
    print(i)
}
```

| 部分 | 说明 |
|------|------|
| `var i = 0` | 初始化语句（声明并初始化循环变量） |
| `i < 100` | 条件表达式（每次迭代前检查，`false` 时退出） |
| `i++` | 更新表达式（每次迭代后执行） |

三个部分都可以省略：

```
// 无限循环
for (;;)
{
    break
}

// 仅条件
var i = 0
for (; i < 10;)
{
    i++
}

// 仅更新
for (var i = 0;; i++)
{
    if (i >= 10)
    {
        break
    }
}
```

### 10.8 Switch 语句 (🔧)

每个 `case` 体必须用 `{}` 包裹，末尾必须显式 `break`（不支持 fall-through）。支持 `when` 条件过滤。

```
switch (value)
{
    case 1:
    {
        print("one")
        break
    }
    case 2:
    {
        print("two")
        break
    }
    default:
    {
        print("other")
        break
    }
}

switch (value)
{
    case x when x > 10:
    {
        print("big")
        break
    }
    case x when x > 0:
    {
        print("positive")
        break
    }
}
```


## 11. 函数

### 11.1 函数声明 (✅)

```
function add(a: int, b: int): int
{
    return a + b
}

function greet(name: string)
{
    print("Hello, " + name)
}

public static Main(): void
{
    // 入口函数，返回 void
}
```

程序入口函数使用 `public static Main(): void`：

```
public static Main(): void
{
    print("Hello World")
}
```

也支持带参数的变体：

```
public static Main(args: string[]): void
{
    print(args[0])
```

返回类型可选，省略默认为 `void`。

### 11.2 函数调用 (✅)

```
add(1, 2)
greet("world")
```

支持嵌套调用：
```
print(add(3, add(4, 5)))
```

### 11.3 函数作为值 (🔧)

```
let op = add          // 函数引用
op(1, 2)              // 通过引用调用
```

## 12. 类与面向对象

### 12.1 类声明 (🔧)

```

{
    private _name: string

    public constructor(name: string)
    {
```

### 12.2 修饰符 (🔧)

| 修饰符 | 类 | 方法 | 字段 | 属性 | 说明 |
|--------|:--:|:----:|:----:|:----:|------|
| `public` | ✅ | ✅ | ✅ | ✅ | 任何代码可访问 |
| `internal` | ✅ | ✅ | ✅ | ✅ | 同一程序集内可访问 |
| `protected` | ❌ | ✅ | ✅ | ✅ | 派生类可访问 |
| `private` | ❌ | ✅ | ✅ | ✅ | 仅本类可访问 |
| `static` | ✅ | ✅ | ✅ | ❌ | 属于类型而非实例 |
| `abstract` | ✅ | ✅ | ❌ | ❌ | 不能实例化/必须实现 |
| `sealed` | ✅ | ✅ | ❌ | ❌ | 不能继承/不能重写 |
| `virtual` | ❌ | ✅ | ❌ | ✅ | 可被派生类重写 |
| `override` | ❌ | ✅ | ❌ | ✅ | 重写基类虚成员 |
| `readonly` | ❌ | ❌ | ✅ | ❌ | 只能在构造函数中赋值 |

### 12.3 继承 (🔧)

```
public class Circle: Shape
{
    private _radius: int

    public constructor(name: string, radius: int): base(name)
    {
        _radius = radius
    }

    public override function Area(): int
    {
        return 3 * _radius * _radius
    }
}
```

- 单继承（一个基类）
- 基类构造通过 `base(...)` 调用
- `override` 重写 `virtual`/`abstract` 方法
- `sealed` 类不能作为基类

### 12.6 `this` 关键字 (🔧)

```
public class Counter
{
    private _count: int

    public function Increment(): int
    {
        _count = _count + 1          // this. 可省略
        return _count
    }

    public constructor() : this(0)
    {
    }

    public constructor(initial: int)
    {
        _count = initial             // this. 可省略
    }

    public function SetCount(count: int)
    {
        this._count = count          // 与参数重名，必须用 this.
    }
}
```

## 22. 命名空间与导入

Cocoa 的命名空间系统用于组织代码并与 .NET 程序集交互。命名空间声明和 `using` 导入仅在编译期影响名称解析。

### 22.1 `using` 导入 (🔧)

`using` 导入将指定命名空间的前缀加入查找路径，使后续代码可以使用非限定类型名称。

```
using System
using System.Collections.Generic

// 非限定名称
List<int> numbers = new List<int>()
Console.WriteLine("hello")
```

### 22.2 命名空间声明 (🔧)

命名空间声明将包含的成员放入指定命名空间：

```
namespace MyApp.Models
{
    public class Customer
    {
        public property Name: string { get; set }
    }
}

namespace MyApp.Services
{
    public class CustomerService
    {
        public GetCustomer(id: int): Customer
        {
            // ...
        }
    }
}
```

### 22.3 完全限定名称 (🔧)

完全限定名称无需 `using`，可直接引用任何命名空间下的类型和成员：

```
System.Console.WriteLine("hello")
System.Math.Max(1, 2)
System.Collections.Generic.List<int>
```

### 22.4 设计要点

- 命名空间声明必须使用 `{}` 包裹成员列表
- 命名空间可以嵌套：`namespace Outer { namespace Inner { } }`
- 文件范围命名空间：`namespace MyApp;`（无需大括号，📋）
- 函数声明在命名空间内时，生成的 IL 类使用 `Namespace.ClassName` 全名
- 编译器依次尝试：直接全名 → import 前缀拼接 → 报错"类型未找到"
- 命名空间不影响运行时行为，仅在编译期影响名称解析
- `using` 关键字双重用途由位置区分：文件级 `using System` 为导入指令（本§）；块级 `using (var r = ...)` 为资源管理语句（见 §10.14）
